# 📱 Protótipo — App Gestantes

## Visão

Transformar Conexão Gestantes de **infoproduto estático** (materiais editáveis em R$297) em **app/plataforma** com **modelo de assinatura**.

## Status

Protótipo em **React web app** construído para validar a ideia e o fluxo. Não é produção — é vitrine para decisão estratégica.

## Módulos do protótipo

| Módulo | Função |
|---|---|
| **Perfis de pacientes** | Cadastro, histórico, dados de gestação |
| **Caderneta virtual de pré-natal** | Versão digital do caderninho tradicional |
| **Histórico de consultas** | Registros por consulta (queixas, exames, condutas) |
| **Rastreamento de exames** | Solicitação + resultados + lembretes |
| **Biblioteca de materiais educativos** | O infoproduto atual, agora como feature |

## Hipótese de modelo

- **Assinatura mensal** para obstetras (substitui ticket único R$297)
- Materiais editáveis + ferramentas de acompanhamento clínico
- Possível camada paciente-facing (caderneta virtual compartilhada)

## Decisões em aberto

- [ ] Ir adiante com produto de assinatura ou otimizar infoproduto atual?
- [ ] Se for adiante: building próprio ou plataforma white-label?
- [ ] Precificação (R$49-99/mês? por médica?)
- [ ] Como migrar os ~350 clientes atuais sem perdê-los?

## Considerações estratégicas

> [!note] Tensão central
> O infoproduto atual funciona. **Não quebrar o que vende** enquanto valida algo maior.
>
> Caminho recomendado de exploração:
> 1. Validar demanda por assinatura com base atual (survey)
> 2. Se positiva, desenvolver MVP com 1-2 módulos
> 3. Oferecer para 10-20 clientes de teste
> 4. Só migrar a operação toda se o sinal for forte

## Relação com tráfego pago

Se App for o produto futuro:
- Reposicionar criativos para "plataforma" e não "materiais"
- Landing vira signup de trial
- LTV projetado maior → permite CPA maior
